# PetaMikke 設計書

**バージョン:** 1.0  
**作成日:** 2026-04-09  
**アプリ名:** PetaMikke（ぺたみっけ）

---

## 1. アプリ概要

### コンセプト
「シールに特化したマーケットプレイス × SNS」

リアル文房具シールの売買・交換・情報共有をワンストップで行えるアプリ。メルカリのような売買機能を軸に、シールレートの共有・販売場所の情報・コレクション自慢など、シール好きが集まるコミュニティを形成する。

### アプリ名の由来
「ペタ（シールを貼る）」＋「みっけ（見つけた！）」

---

## 2. ターゲットユーザー

| セグメント | 年齢層 | ニーズ |
|---|---|---|
| メインターゲット | 小学生〜中学生 | シール交換相手を探したい、レートを知りたい |
| サブターゲット | 20〜30代女性（Z世代・平成女児） | コレクション管理、SNS映えしたい |
| 保護者 | 30〜40代 | 子どものシール活動を安全に管理したい |

---

## 3. 主要機能一覧

### 3.1 マーケット機能（メルカリベース）
- シールの出品・購入
- シール交換の申し込み（リアル交換マッチング）
- レート表示付き価格提案
- カテゴリ・シリーズ別検索
- ウォッチリスト（お気に入り登録）

### 3.2 SNS・コミュニティ機能
- シール帳の投稿・公開（写真共有）
- いいね・コメント・リポスト
- ハッシュタグ（#ボンドロ、#うるチュル など）
- フォロー・フォロワー
- ストーリー形式の「今日の収穫」投稿

### 3.3 シール情報データベース
- シール種類の百科事典（ボンドロ・うるチュルなど詳細情報）
- コミュニティ投票によるレートランキング
- 入荷情報・販売場所の共有（マップ連携）
- 偽物注意情報の共有

### 3.4 コレクション管理
- 所持シールの登録・管理
- 「持ってる」「欲しい」「交換可能」タグ付け
- シール帳のデジタル再現（ページレイアウト自由配置）
- 写真スキャン → AI自動認識・登録（将来機能）

### 3.5 交換マッチング
- 「交換したい」リスト公開
- 相互マッチング通知（お互いが欲しいシールを持っている場合）
- 交換場所（店舗・公共施設）の提案
- 交換完了レビュー・評価

## 5. データモデル（ER図）

### 主要エンティティ

```
User（ユーザー）
├── user_id
├── username
├── display_name
├── avatar_url
├── age_group（子ども / 大人）
├── rating（交換評価スコア）
├── created_at

Seal（シール）
├── seal_id
├── name（シール名）
├── series（シリーズ：ボンドロ / うるチュルなど）
├── type（立体 / ペラ / ウォーター etc.）
├── character（キャラクター：サンリオ / ちいかわ etc.）
├── base_rate（基準レート）
├── image_url
├── created_at

UserSeal（ユーザー所持シール）
├── user_seal_id
├── user_id → User
├── seal_id → Seal
├── quantity（枚数）
├── condition（状態：未使用 / 良好 / 傷あり）
├── status（所持 / 交換可 / 欲しい）

Listing（出品）
├── listing_id
├── seller_id → User
├── seal_id → Seal
├── price（円）または trade_rate（交換レート）
├── type（販売 / 交換 / 両方）
├── quantity
├── description
├── status（出品中 / 取引中 / 完了）
├── created_at

Trade（交換申し込み）
├── trade_id
├── requester_id → User
├── responder_id → User
├── offer_seal_id → Seal（提供するシール）
├── want_seal_id → Seal（欲しいシール）
├── status（申込中 / 合意 / 完了 / キャンセル）
├── created_at

Post（投稿）
├── post_id
├── user_id → User
├── type（シール帳 / 収穫報告 / 情報共有）
├── content（テキスト）
├── images（複数画像URL）
├── hashtags
├── likes_count
├── created_at

SealLocation（販売場所）
├── location_id
├── name（店舗名）
├── type（文房具店 / コンビニ / 100均 / 雑貨店など）
├── latitude / longitude
├── reported_by → User
├── last_confirmed_at（最終確認日）
├── available_series（取扱シリーズ）

Review（評価）
├── review_id
├── trade_id → Trade または Listing
├── reviewer_id → User
├── reviewee_id → User
├── score（1〜5）
├── comment
```

## 6. レート（価値基準）システム

### レートの定義
シール1枚あたりの相対的な価値。コミュニティ投票と出品データをもとに自動算出。

### レートランキング（2026年4月時点）

| ランク | シール種類 | 相対レート |
|---|---|---|
| S | ボンボンドロップシール（大） | 8 |
| A+ | ボンボンドロップシール（小） | 5 |
| A | うるチュルポップシール | 4 |
| A | プチドロップシール | 4 |
| B | マシュマロシール | 3 |
| B | ウォーターシール | 3 |
| C | タイルシール | 2 |
| D | ペラシール（通常） | 1 |

### レート更新ルール
- 週次でコミュニティ投票集計
- 出品価格の中央値を参照
- 急激な変動は運営が確認

## 7. 技術スタック（案）

### フロントエンド
- **フレームワーク:** Flutter（iOS / Android 同時対応）
- **状態管理:** Riverpod
- **デザインシステム:** Material Design 3 ベース（カスタムテーマ）

### バックエンド
- **API:** Supabase（認証・DB・ストレージ・リアルタイム）
- **画像ストレージ:** Supabase Storage
- **マップ:** Google Maps API

### 通知・メッセージ
- **プッシュ通知:** Firebase Cloud Messaging
- **チャット（交換交渉用）:** Supabase Realtime

### 将来的な拡張
- AI画像認識（シール自動登録）: Google Vision API
- レコメンドエンジン: Cloud Functions

## 8. 収益モデル

| 収益源 | 内容 | 手数料 / 料金 |
|---|---|---|
| 販売手数料 | シール販売時の仲介手数料 | 販売価格の10% |
| プレミアム会員 | 広告非表示・コレクション無制限登録 | 月額300円 |
| 広告 | シール関連商品の広告表示 | CPM |
| 公式販売店舗掲載 | 店舗の優先掲載・入荷速報 | 月額定額 |
